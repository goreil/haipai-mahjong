# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What This Is

Personal Riichi Mahjong game analysis workspace. Contains game review logs from Tenhou replays analyzed with the Mortal AI, learning notes, and a summary calculator script.

## Key Files

- `Mahjong Mistakes.txt` - Game review log tracking mistakes by category/severity with EV loss.
- `Richii Mahjong Learning.txt` - Strategy notes: defense heuristics, tile efficiency rules, when to open/fold.
- `mj_parse.py` - Parses Mortal AI JSON analysis into skeleton game entries (turn numbers + EV loss).
- `mj_calc.sh` - Main script: calculates summaries and manages game entries.
- `mj_summary.sh` - Standalone summary calculator (legacy, superseded by mj_calc.sh).
- `mahjong-cpp/` - Git submodule ([goreil/mahjong-cpp](https://github.com/goreil/mahjong-cpp)), a C++ mahjong library.

## Commands

```bash
python3 mj_parse.py analysis.json          # Extract mistakes from Mortal JSON to stdout
python3 mj_parse.py analysis.json --date 2026-03-25  # Override date
./mj_calc.sh calc    # Recalculate SUMMARY/TOTAL/TURNS for all games (default)
./mj_calc.sh new     # Append a new blank game entry with today's date
```

## Game Log Format

Games in `Mahjong Mistakes.txt` are delimited by `======` lines. Each game has:
- `Date:` and `Log:` header lines
- Round headers: `E1T13` (East 1, 13 turns), `S2-1T14` (South 2 continuation, 14 turns). Turn count (`TN`) is optional.
- Round headers may have outcome markers: `:D` (great), `:)` (good), `:|` (neutral), `:(` (bad).
- Mistake lines: `Turn Severity Category EV (note)`

Severity levels: `!` (flagged/notable), `?` (0-0.50 EV), `??` (0.50-1.00 EV), `???` (>1.00 EV).

Categories: 1A-1E (efficiency/dora/honors/pairs), 2A-2C (defense), 3A-3C (melding), 4A-4B (riichi), 5A-5B (kan).

`SUMMARY:`, `TOTAL:`, and `TURNS:` lines are auto-generated by `mj_calc.sh calc`. `TURNS:` only appears when round headers include turn counts.
