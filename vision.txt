My vision for this is having a neat trainer tool to help me
study my mahjong mistakes. My current workflow is:

1. Play a game
2. Paste the game link into the mortal analysis engine
Note: This must be done by hand, since it is protected by cloudflare
3. Run: python3 mj_games.py add '<mjai-viewer-url>'
   This fetches the JSON, parses all mistakes with full hand/discard data,
   and stores everything in games.json.
4. Review with: python3 mj_games.py review --game N
5. Annotate categories/notes: python3 mj_games.py annotate ...
6. View summaries: python3 mj_games.py summary

The grand vision would be to build a web-app that I may even
make some money with since there are currently
no good english apps like that online.


=== COMPETITIVE LANDSCAPE (researched 2026-03-31) ===

What English riichi players have today:
- Mortal (mjai.ekyu.moe): free per-turn AI review, but raw viewer only. No categorization,
  no practice mode, no trend tracking, no "why" explanation.
- NAGA: paid replay analysis, but needs Nicovideo account + Japanese knowledge. Inaccessible
  to most English speakers.
- Akochan Reviewer: CLI tool, static HTML output, no web UI, no taxonomy.
- Euophrys' Efficiency Trainer (itch.io): generic random hands, not your own games.
- Riichi Scoring Trainer, Waits Trainer: specific skill drills, no game review.

What nobody has:
- Mistake categorization across a taxonomy (our 15-category system)
- Practice quizzes from your own past errors
- Trend analytics over time (EV/decision progression)
- Mortal-vs-mahjong-cpp comparison to classify WHY a mistake happened

Differentiation story: Mortal tells you WHAT you did wrong. This app tells you WHY,
tracks it over time, and lets you drill your actual weak spots.


=== DONE (summary) ===

Core pipeline: CLI (add/review/annotate/summary/list/categorize), JSON->SQLite data store,
Mortal JSON parser, auto-categorization (mortal vs mahjong-cpp), 15-category taxonomy.

Web UI: Flask app with game list, review view (SVG tiles, severity colors), inline annotation,
severity filters, add-game modal, positive game ratings (star system).

Analysis: EV comparison tables (Mortal Q vs mahjong-cpp), defense analysis (suji safety 0-15),
2A/2B defense-aware split, trend charts (EV/decision, severity, skill area), board state view.

Practice mode: discard quizzes from own mistakes, clickable tiles, filters (category/severity/
riichi/calc-agrees), spaced repetition, session scoring, keyboard shortcuts.

Multi-user: Docker + nginx + certbot, SQLite with user isolation, Flask-Login,
invite-code registration, branding ("Haipai"), security hardening, feedback form.

Testing: pytest suite (21 tests), local-only nanikiru server.

See git log for detailed history. Archived details in archive/.
